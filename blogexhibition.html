This is a project I worked as a solo program developer with designers. We aimed to create involving experience in a graduation project exhibition. Audience could use their phone to check briefs of each project and plan their routes quickly. There would be an electronic voting button beside each exhibition booth. Once the audience found something interesting and push down the voting button, they would see the changes in popularity immediately from the webpage. The responsive webpage shall be able to embedded in WeChat and display on monitor on-site.

DESIGN

Collaboration with Visual Designers

This web tool serves functions:
1. Show exhibition map
2. Help user plan route based on interests
3. Link similar projects — “you may also like"
4. Navigate to VR page
5. Disaplay vote ranking in real time
Designers worked on visual design first. I started with a bunch of design drawings. We worked on interactive prototyping on proto.io together, designed navigation and transition effect together.
We sent our prototype to other students to play around. It helps us discover some interesting facts of web interaction.
1. We initially wanted to use slide gesture to show a floating section. But after usability test, we found when user slide to scroll page, it causes slip to slide up another section. So we changed the gesture to tap.
2. When the color opaque of location icon is low, user couldn’t recognize it is clickable, so to miss the map function.
I also learnt valuable lessons from visual designers. For example, they are really good at arranging colors, sounds and interaction effects into the theme of whole product. They taught me some of their stylish theories and how to use tools like Adobe AI.

CODE

Build Webpage on a Server

It was my first time to deploy a server for usage in a real event where performance issue really gets critical rather than a word in documentation. When some functions consume too much performance, we had to find another effective technology to realized the function. This project contains much dynamically generated content, including exhibition item introduction and map, front-end and back-end interaction to devise and save personalized route.
I was glad to have this self-exploratory experience in web development. Although due to time limitation, I only digested a small amount of knowledge which are needed in this project, I had a general understanding of those great popular tools — AJAX, Gulp, React.JS, NGIX, NodeJS, and I could make wiser selection in my next web project. I am also very grateful for the explicit documents and helpful community of those tools — I am definitely looking forward to more web work;)
